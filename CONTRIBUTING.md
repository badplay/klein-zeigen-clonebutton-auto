# Beitragen

Danke für dein Interesse an diesem Projekt! Da es sich um ein **persönliches Test-Projekt** handelt, sind Beiträge willkommen, aber es gibt keine Garantie auf zeitnahe Reviews.

---

## Wie kann ich beitragen?

### 🐛 Fehler melden

1. Schaue zuerst in den [Issues](../../issues), ob der Fehler bereits gemeldet wurde.
2. Falls nicht, erstelle ein neues Issue mit:
   - Browser und Version
   - Tampermonkey-Version
   - Beschreibung des Fehlers
   - Schritte zur Reproduktion
   - Screenshot oder Konsolenausgabe (wenn möglich)

### 💡 Verbesserungen vorschlagen

Erstelle ein Issue mit dem Label `enhancement` und beschreibe deine Idee.

### 🔧 Pull Requests

1. Forke das Repository
2. Erstelle einen neuen Branch: `git checkout -b feature/meine-funktion`
3. Mache deine Änderungen
4. Teste die Änderungen im Browser
5. Committe: `git commit -m "feat: Beschreibung der Änderung"`
6. Push: `git push origin feature/meine-funktion`
7. Erstelle einen Pull Request

---

## Coderichtlinien

- Kommentare auf **Deutsch** (da das Projekt für deutschsprachige Nutzer ist)
- Keine externen Abhängigkeiten im UserScript
- Änderungen in `CHANGELOG.md` dokumentieren
- Version in der UserScript-Kopfzeile aktualisieren
